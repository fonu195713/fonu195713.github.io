    橋接模式 #

    • ChatGPT 的說明

    橋接模式（Bridge Pattern）是軟體設計模式的一種，屬於結構型模式，該模式用於將程式中抽象部分與其實例物件分離，另它們可以獨立變化而互不影響。橋接模式主要旨在解決軟體設計中的類別膨脹問題：意即隨著新功能的增加，系統中的類別數量容易呈指數的量級增長，進而難以對系統進行維護和擴展。在橋接模式中，有兩個重要的概念：分別是【抽象部分（Abstraction）】和【實例物件（Implementor）】。【抽象部分】通常會定義一個（或多個）不同的程式接口，包含了客戶端所期待的行動或操作，並且保存了一個（或多個）實例變數來引用【實例物件】；【實例物件】則是抽象部分的實例化結果，該物件會實現抽象部分所定義的接口，並完成其功能。

    橋接模式的核心思想是將抽象部分和實例物件分開，讓它們可以獨立變化，從而使得系統更具靈活性和可擴展性。通過橋接模式，我們可以將系統中，多個維度的複雜節狗給區分開來，並通過組合這些維度的不同物件來達到多樣化的效果，同時又不會產生類別膨脹的問題，增加程式的可讀性、可維護性、與使用效率。

    • 簡單整理

    問題描述：若系統中的多項成員變數都有多種改變，建置所有可能組合的類別群會是個不好的方法

    解決方案：將參數抽象化，使得實例化的物件可以自由組合不同的成員變數細節
