    原型模式 #

    • ChatGPT 的說明

    原型模式（Prototype pattern）是一種創建型設計模式，其主要目的是『透過不使用建構子的方式，創建類別中的新物件。』此模式的核心思想是基於已有對象的拷貝（拷貝）來產生新的對象，這個已有對象稱為原型，而拷貝的方式有淺拷貝（Shallow clone）和深拷貝（Deep clone）兩種。原型模式有以下關鍵特點和優點：［透過 clone 創建物件］：原型模式允許程式在運行的時後創建新的物件，而無需知道特定物件的構造細節。這是透過複製現有物件的屬性和狀態來實現的。［減少構造成本］：在某些情況下，物件的構造可能非常昂貴（像是需要花費大量資源、消耗大量時間 …等），透過原型模式，可以複製已有物件，來避免重複執行構造操作，從而提高效能和效率。［支援多樣化的物件］：原型模式允許建立多個不同屬性的物件，而無需重新實作整個物件的建構邏輯，這對於物件的變種非常有用。［簡化物件的創建］：原型模式簡化了物件的創建過程，尤其是在物件具有複雜結構或大量屬性時。可以使客戶端只需要呼叫模式中的 clone 方法，便可以實踐物件的創立。

    而在原型模式中，拷貝（或拷貝）物件的方式總共分成兩種：［淺拷貝］（Shallow clone）：淺拷貝會在方法內建立一個新原型的物件，但該新物件的內部成員變數仍然與原始的原型物件共用同樣的子物件。這意味著，雖然新物件本身是與原物件獨立的，但在新物件內部仍然引用了原始物件中的某些子物件。如果對我們對任一物件的共享去性進行修改，這些修改將在兩個物件中都可見，因為它們引用同一份資料。淺拷貝通常是較快且較節省記憶體的方式，但需要注意在共享子物件上可能出現潛在問題。［深拷貝］（Deep Copy）：深拷貝重樣會創建一個新的物件，但與淺拷貝不同的是。這種「創建」包括原始的原型物件，以及該物件的所有子物件（以遞回的方式完全複製整個物件結構）。這確保了新物件與原始物件之間完全獨立，對其中一個物件所做的修改不會影響另一個物件。深拷貝通常比較安全，因為它避免了共享資料的問題，但可能需要更多時間和記憶體來執行，特別是對於複雜的物件結構。

    在這樣的情境下，透過單純「對 query 語句進行極致的優化，已達成『不中斷』的需求」就不是一個很好的辦法。因此，為了解決目前的問題，我們就可以應用到本篇文章在討論的原型模式（Prototype pattern）。原型模式，是一種「透過物件本身的 clone() 方法快速建立新物件」的一種程式撰寫技巧。透過單純複製現有物件的參數內容，省去大量與其他系統溝通的 IO 時間（例如與資料庫系統的溝通）。進而達到「加速」的需求。

    import java.time.LocalDateTime;
import java.util.ArrayList;
import java.util.List;

class Member implements Cloneable {
    private int memberId;
    private LocalDateTime joinDate;
    private List<Order> orders;

    public Member(int memberId, LocalDateTime joinDate) {
        this.memberId = memberId;
        this.joinDate = joinDate;
        this.orders = new ArrayList<>();
    }

    // Member 的其他方法與 getters, setters ...

    // 建立一個 clone() 方法，使物件本身允許被複製
    @Override
    protected Object clone() throws CloneNotSupportedException {
        Member clonedMember = (Member) super.clone();
        clonedMember.orders = new ArrayList<>();

        // 遞迴複製每個 Order 物件
        for (Order order : this.orders) {
            clonedMember.orders.add((Order) order.clone());
        }

        return clonedMember;
    }
}

    import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.sql.SQLException;

class Database {

    private static final String PASSWORD = "root";

             Statement statement = connection.createStatement();

            while (resultSet.next()) {
                Member member = new Member(resultSet.getInt("member_id"),
                        resultSet.getTimestamp("join_date").toLocalDateTime());

                // 先把資料暫存在變數 members 裡面
                members.add(member);
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }

    public static Database getInstance() {
        return DatabaseHolder.INSTANCE;
    }

    public List<Member> getMembersFromMemory() {
        return members;
    }

        LocalDateTime targetDateTime = LocalDateTime.of(currentDateTime.getYear(), Month.NOVEMBER, 11, 0, 0);

        while(true) {
            LocalDateTime currentDateTime = LocalDateTime.now();

            if (targetDateTime.isBefore(currentDateTime)) {
                // 拿出原本就暫存在 database 的 members
                // 省去需要跟 DB system 溝通的時間

                for (Member member : members) {
                    List<Order> orderList = member.getOrders();

                        LocalDateTime lastOrderDate = latestOrder.getOrderDate();

                        long years = lastOrderDate.until(currentDateTime).toDays() / 365;

                        if (years < 2) {
                            member.setCoin(member.getCoin() + 1111);
                        }
                    }
                }

                break;
            }

            try {
                // 每秒檢查一次
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
