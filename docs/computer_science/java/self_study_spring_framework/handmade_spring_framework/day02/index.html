    Day02 - bean and Collections #

    Day 02, 筆記 #

    • 【前言】

    雖然說是 day 2，但實際上距離 day 1 應該已經過去 3-4 天了。沒什麼特別的原因，純粹就只是自己比較懶惰所以沒有快速補進度這樣（有點不好… 以後希望可以再努力一點）。總之 day 2 的內容基本上是延續 day 1 再往下的，大致上沒有什麼程式上面的新語法，更多的是注重在 xml 上面的… 物件宣告(?) 也就是各種 <bean> 標籤的生成語法，以及物件的賦值，包含基本資料型態（int, float, char, boolean…）以及參考資料型態（String, Object, Class…），還有集合類型（Array, List, Map …）的宣告。

    • 【結果】（成功）

    • 【過程】

    相較於第 1 天的內容，第 2 的內容算是簡單很多，因為主要都是在介紹一些宣告方式、物件的賦值方法、以及 xml 標籤的一些介紹或定義之類的。程式碼的部分相對少了很多，取而代之的是一系列的、五花八門的、各式各樣不同的物件（<bean>）宣告方式，但在講有哪些做法之前，先說明一下這次的檔案結構：為了自己的方便理解，我稍微複雜化了原本的檔案結構，主要是在 human 資料夾裡面做一些更動，圖示如下：

    主要改動有兩個：第一個是 human 資料夾的內容從原本的 Person 變成了 Human, Student 以及 Teacher 三種類別。Human 是學生跟老師的父類別，該類別裡面只有一個成員變數 name，以及它的 setter setName()；學生（Student）繼承自 Human，除了擁有姓名之外，還多擁有一個分數 score 的變數，用來記錄不同科目（Subject）的考試分數；最後是老師 Teacher，同樣繼承自 Human 因此擁有姓名，此外老師也有一份學生名單 List<students> 用來存放該老師是負責哪幾位學生的。

    然後是 subject 資料夾，裡面有 4 份檔案，分別是 Subject, Chinese, English 以及 Math 四個。其中 Subject 是其餘三個類別的父類別，裡面有一個 name 的成員變數（用來記錄科目的名稱），然後就沒有其他東西了。剩餘的三個科目裡面基本沒有程式內容，就是單純的繼承以及設定名字這樣。程式如下：

    package human;

public abstract class Human {
    protected String name;

    public void setName(String name) {
        this.name = name;
    }
}

    • 【bean】

    解釋完檔案結構跟內容之後，接下來就可以進入到 day 2 的主軸了。前面說過，day 2 的主要內容在於各種不同的 <bean> 標籤的物件生成和賦值方式，主要有「setter 賦值」「建構子創建」「p-namespace 創建」「ref 參考賦值」「p-namespace-ref」「內部 bean」「工廠模式」「生成單例物件」和「集合類型的創建」幾種不同的物件生成和賦值方式。

    • setter 賦值

    做為最常見、最泛用的一種賦值方式，setter 賦值的做法很簡單，先透過 <bean> 標籤生成一個物件（該物件要有不帶任何參數的建構子），再在 <bean> 標籤內部放入一個或多個 <property name="" value=""> 標籤對物件的成員變數進行賦值即可，其中 name 是指 setter 的名稱，value 則是指要帶進去的參數值，像是以下的標籤，就是「生成一個 “human.Student 類別的物件，並將該物件的 name 屬性設定為 student1」：

    <!-- generated through the <property> tag ((most used)) -->
<bean id="s1" class="human.Student">
    <property name="name" value="student1"></property>
</bean>
    • 建構子創建

    除了用『不帶任何參數的建構子』建構物件之外，我們也可以使用帶有參數的建構子進行物件的創建。我們會需要用到一個 <constructor-arg> 標籤，裡面需要包含三種屬性，分別是 index（代表該參數放在第幾個位置）、type（代表參數類別）、以及 value（代表參數值）。需要注意的是 constructor 是單標籤，所以所有的屬性寫在標籤裡面就可以了，不需要用包的：

    <!-- generated through the constructor -->
<bean id="s2" class="human.Student">
    <constructor-arg index="0" type="java.lang.String" value="student2"/>
</bean>

    直得注意的是：如果今天 Student 的建構子長這樣：

    class Student {
    public Student(Integer age, Student Name, Double height) {}
}

    那標籤就可以寫成下面這樣：

    <!-- generated through the constructor -->
<bean id="s2" class="human.Student">
    <constructor-arg index="0" type="java.lang.Integer" value="16"/>
    <constructor-arg index="1" type="java.lang.String" value="小明"/>
    <constructor-arg index="2" type="java.lang.Double" value="172.4"/>
</bean>
    • p-namespace 創建

    p-namespace 算是 <bean> 配合 <property> 裡面一個比較簡潔(?) 的寫法，需要先在文件最上面的 <beans> 命名空間中加入 xmlns:p 的 p-namespace 語法宣告，然後就可以用了：可以把原本 <property> 中 name 的內容寫成 p:"name"，然後後面接上參數值。整體邏輯跟最上面的 setter 一樣，只是視覺上清爽了許多：

    <!-- 最上方的命名空間 -->
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:p="http://www.springframework.org/schema/p"
    xsi:schemaLocation="
        http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans-3.0.xsd>

    <!-- 使用 p-namespace 之前 -->
<bean class="human.Student">
    <property name="name" value="小華"></property>
    <property name="age" value="13"></property>
    <property name="height" value="163.7"></property>
</bean>

<!-- 使用 p-namespace 之後 -->
<bean class="human.Student"
    p:name="小華" p:age="13" p:height="163.7">
</bean>
    • ref 參考賦值

    ref 適合用在「當參數值不是基本資料型態的時候」，可以寫成 <property ref="..."> 的形式，來讓物件去吃一個物件型別的參數（ref 中的內容需要是 bean 的 id）。舉例來說：如果「班級」裡面有「老師」以及「班級名稱」這兩個不同的成員變數，那麼我們就可以寫成像下面的這個樣子：

    class Classroom {
    public Classroom() {}

    public void setName(String name) {
        this.name = name;
    }

    public void setTeacher(Teacher teacher) {
        this.teacher = teacher;
    }
}

    <bean id="t1" class="Teacher">
    <property name="name" value="陳老師"></property>
</bean>

<bean class="Classroom">
    <property name="Teacher" ref="t1"></property>
    <property name="name" value="一年６班"></property>
</bean>
    • p-namespace-ref

    p-namespace-ref 的寫法基本上是 p-namespace 加上 ref 的組合：

    <bean id="t1" class="Teacher">
    <property name="name" value="陳老師"></property>
</bean>

<bean class="Classroom"
    p:Teacher="t1" p:name="一年６班">
</bean>
    • 內部 bean

    內部 bean 的寫法也很好理解：property name=“Teacher” 指的是「這項參數會被丟到名為 setTeacher() 的方法內」，而 property 裡面再放一個 bean，用來建構 Teacher 的類別，這樣就不用再透過設定 bean id，然後 ref="..." 的方式指定物件了。只不過內部 bean 的內容只有該物件本身可以使用，物件以外的其餘類別或物件都不可以去存取這個內部 bean：

    <bean class="Classroom">
    <property name="name" value="一年６班"></property>
    <property name="Teacher">
        <bean class="Teacher">
            <property name="name" value="陳老師"></property>
        </bean>
    </property>
</bean>
    • 工廠模式

    工廠模式是透過一個 facroty 生成 bean 的方法。需要在 Java 檔案中的類別實例化一個名為 FactoryBean<T> 的介面。該介面主要包含三種方法，分別是用以取得物件的 getObject() 方法、用以辨識物件類別的 getObjectType() 方法，以及判斷該類別是否為單例的 isSingleton() 方法。只要在 Java 中實例 FactoryBean<T> 的介面，就可以透過 FactoryBean 創建新的物件。

    • 生成單例物件

    在某些時候，我們會期望某一個類別只能生成唯一的一個物件，這時候就可以用單例的寫法：我們假設 Classroom 這個類別在整個程式執行的過程中，只可以有一個。那這樣的話我們就可以用下面的寫法（scope="singleton"）來確保 Classroom 的類別只會出現一個實例物件：

     <bean id="classroom" class="Classroom" scope="singleton">
    <property name="name" value="一年６班"></property>
</bean>
    • 集合類型的創建

    最後是集合類型的創建，集合類型就是泛指 List, Array, ArrayList, Map, Set … 等各種不同的 Java 容器類別。在創建容器類別的物件之前，需要先在最上方的 <beans> 命名空間添加 xmlns:util 的宣告，然後我們就可以利用諸如 util:list, util:map, util:set … 等等的標籤來宣告/生成容器物件了。容器內可以放原生資料型態（int, double, char…等）也可以放參考資料型態。參考資料型態同樣可以用 ref 的方式嵌入，或者是用內部 bean 放置皆可。
