    Day01 - Simple Core Container #

    Day 01, 筆記 #

    • 前言

    整個 Spring 的模組大概就是上面的那些。扣掉最底層的 Test 層(?)，整個 Spring 的框架基本上是被 Core Container 裡面的 Beans, Core, Context, 和 SpEL 這四個東西所建立起來的，所以今天的目標就是嘗試只用這 4 個東西，去做一個簡單的程式出來。對於什麼功能上的要求都先忽略，只要先確定程式能夠跑起來就好。所以今天的目標是：透過 Spring 裡面的這四個東西，做出一個 Hello world 的程式來。

    • 結果（成功）

    • 過程

    整個過程可以說是非常克難… 幾乎可以理解成重新學了一次 Java 從編譯到執行的流程。首先的第一個工作是「把 Spring 的 jar 檔丟給我要的程式吃」。因為在此之前並沒有學習到關於 jar 檔案的觀念，所以算是理解了一次 .jar, .java, .class 三者之間的關係，也複習和新學到了 javac, javap, jar, java 等不同的 cmd 指令的用途。這裡算是第一個在學習 Spring 的過程中，有收穫新東西的地方。

    程式整體的專案結構如上圖，所有的檔案都被放在一個「day01」的資料夾裡面，然後 day01 裡面只有一支 TestByBean.java，作為整個專案的…入口？然後 day01 裡面還有 3 個資料夾，分別是 conf, human, 和 spring。conf 裡面放的是設定檔，設定什麼的等等會說，human 裡面有一個 Person 的檔案，是很簡單的類別內容，在專案中嚴格來說並不需要這樣割（可以把 Person 跟 TestByBean 放在同一層），但因為我想練習一下 package 的語法，就這樣切了。

    package human;

public class Person {
    private String name;
    private Integer age;

    public Person() {}

    public void setName(String name) {
        this.name = name;
    }

    public void setAge(Integer age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Person [name=" + this.name + ", age=" + this.age + "]";
    }

    public void greeting() {
        System.out.println();
        System.out.print("Hello! My name is " + this.name + ", ");
        System.out.println("I'm " + this.age + " years old.");
    }
}

    然後是 spring 的資料夾，裡面有 5 個不同的 .jar 檔，分別是架構圖上，Core container 裡面的字個模組（Bean, core, Context, 和 SpEL（SpEL = expression））以及一個 logging 檔，用來生成一些 console 的 log。這邊的重點是需要「把這些 jar 餵給最外層的 TestByBean」，所以就會用到 javac -cp 這個東西，也算是小小的碰撞學習（不能用 *.jar 要用 *、不加 . 吃不到當前路徑…等）。

    javac -cp ".:springs/*:human/*" TestBySpring.java
java -cp ".:springs/*:human/*" TestBySpring

    把檔案丟給程式吃之後，接下來要透過這些 Spring 的檔案跑一次簡單的程式。本質上，這份專案是一個很簡單很簡單的程式，主要由 TestByBean.java 這個檔案作為專案的主要入口，所以 main() 是寫在這邊。Spring 的概念有點像是相對於「直接使用 new」生成物件，改而透過所謂的「容器」來幫助、管理 Java 中的各種物件。在這支程式中，容器指的就是 ac，也就是第 9 行程式的內容。

    ApplicationContext ac = new ClassPathXmlApplicationContext("conf/applicationContext.xml");

    容器在產生之前，會需要先吃一個設定檔（configuration file），這邊的設定檔就是參數裡面的 xml 檔。接下來把 new 出來的容器放在名為 ac 的變數上，最後透過設定檔本身的內容，讓 ac 用一個 getBean 去抓一個 Person 物件，最後讓那個 Person 做事情（say hello），就是整隻程式裡面在做的事情了。值得注意的事情是：程式本身並沒有什麼比較特別的內容，但 xml 裡面就有一些可以說的東西了：

    <!-- applicationContext.xml -->

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans-3.0.xsd">

    <bean id="person" class="human.Person">
        <property name="name" value="Ming"></property>
        <property name="age" value="15"></property>
    </bean>
</beans>

    最外層的 beans 標籤是固定寫法，為什麼裡面會有那些內容目前還在研究中，但總之可以知道的是：如果要讓 Spring 正常運作的話，就必須先這樣寫，我目前的想法是把這個 beans 當成所謂的「容器」。容器裡面目前只包了一個 bean 標籤（沒有 s），基本上，可以把一個 bean 當作是一個「物件」，所以這個設定檔本質上，就是在描述「一個只放了一個物件在裡面的容器」，我是這麼理解的，可能有誤？但沒關係，先記憶起來。

    該物件的類別是 human.Person，也就是那個被我拉進 human 資料夾裡面的 Person，該物件的 id 是 person，這個 id 可以自己隨便取，叫做小明 id="ming" 小華 id="hua" 或者是小美 id="mei" 之類的都行，但跟 html 一樣，整份檔案裡的 id 必須要唯一，不可重複。此外，物件裡面包含了兩個的 property，也就是 … 屬性？用以對物件做一些初始賦值，裡面的 name 是 getter 的名稱，value 則是要丟進去的值，因此 <property name="name" value="Ming"> 這句話的意思就是：丟入參數 Ming 到 human.Person 類別裡面的 setName 方法中。<property name="age" value="15"> 同理，就是把 15 丟進 setAge 的方法裡面。那因為是這樣的邏輯，所以如果 Person.java 跟 xml 改成下面這樣 …

    public class Person {
    private String name;

    public Person() {}

    public void setAsd(String name) {
        this.name = name;
    }
}

    <!-- applicationContext.xml -->

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans-3.0.xsd">

    <bean id="person" class="human.Person">
        <property name="asd" value="Ming"></property>
    </bean>
</beans>

    程式其實還是可以順利運行，但極度反直覺就是了。
